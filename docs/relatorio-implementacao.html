🎯 1. Visão Geral do Projeto

O sistema CTG Engenharia é uma aplicação full-stack composta por:

Componente Tecnologia Versão Descrição
Backend Node.js + Express 2.0.0 API RESTful com autenticação JWT
Frontend React + Vite 2.0.0 Interface web com PWA
Banco de Dados PostgreSQL 14+ Armazenamento persistente
ℹ️ Nota: A aplicação pode ser deployada de duas formas: instalação direta (Node.js + PostgreSQL) ou via Docker containers.

🔧 3. Método 1: Instalação Direta (Node.js + PostgreSQL)

1 Instalar Node.js v22 (LTS)

Acesse https://nodejs.org e baixe a versão LTS (v22).

# Verificar a instalação
node --version
# Deve retornar: v22.x.x

npm --version
# Deve retornar: 10.x.x ou superior
2 Instalar PostgreSQL 14+

Instale o PostgreSQL seguindo as instruções oficiais para seu sistema operacional.

# Verificar a instalação
psql --version
# Deve retornar: psql (PostgreSQL) 14.x ou superior
3 Configurar o Banco de Dados

Crie um banco de dados para a aplicação:

# Acessar o PostgreSQL
sudo -u postgres psql

-- Criar banco de dados
CREATE DATABASE ctg_engenharia;

-- Criar usuário (opcional)
CREATE USER ctg_user WITH PASSWORD 'sua_senha_segura';
GRANT ALL PRIVILEGES ON DATABASE ctg_engenharia TO ctg_user;

-- Sair
\q
4 Instalar Dependências do Backend 
# Navegar para a pasta do backend
cd backend

# Instalar todas as dependências
npm install

Dependências principais:

  • express (framework web)
  • pg (driver PostgreSQL)
  • jsonwebtoken (autenticação JWT)
  • bcrypt (criptografia de senhas)
  • helmet (segurança)
  • cors (política de origem cruzada)
  • multer (upload de arquivos)
  • exceljs (manipulação de Excel)
5 Instalar Dependências do Frontend 
# Navegar para a pasta do frontend
cd frontend

# Instalar todas as dependências
npm install

Dependências principais:

  • react + react-dom (UI framework)
  • vite (build tool)
  • axios (cliente HTTP)
  • react-router-dom (roteamento)
  • recharts (gráficos)
  • zustand (gerenciamento de estado)
6 Configurar Variáveis de Ambiente

Crie um arquivo .env na pasta backend com o seguinte conteúdo:

# backend/.env
DATABASE_URL=postgresql://ctg_user:sua_senha@localhost:5432/ctg_engenharia
PORT=3001
NODE_ENV=production
JWT_SECRET=seu_segredo_jwt_aleatorio_e_longo_aqui

# Seed (opcional — valores padrão se não definir)
ADMIN_NAME=Administrador CTG
ADMIN_EMAIL=admin@ctgbrasil.com
ADMIN_PASS=ctg@2026
⚠️ Importante: Altere o JWT_SECRET para um valor aleatório e longo. Não use o valor de exemplo em produção!
7 Build do Frontend 
# Navegar para a pasta do frontend
cd frontend

# Gerar build de produção
npm run build

# Copiar o build para o backend (se necessário)
cp -r dist ../backend/public

🐳 4. Método 2: Deploy com Docker

ℹ️ Nota: Este método é mais simples e recomendado para ambientes de produção, pois isola completamente a aplicação.
1 Instalar Docker Engine

Siga as instruções oficiais em https://docs.docker.com/engine/install/

# Verificar a instalação
docker --version
# Deve retornar: Docker version 20.10.x ou superior
2 Configurar Arquivo .env

Crie o arquivo .env na raiz do projeto com as variáveis necessárias (veja seção 6).

3 Subir a Aplicação 
# Navegar para a pasta raiz do projeto
cd forecast

# Build e execução (se houver docker-compose.yml)
docker-compose up -d

# OU usando o Dockerfile diretamente
docker build -t ctg-engenharia .
docker run -d -p 8080:8080 --env-file .env ctg-engenharia
✅ Pronto! A aplicação subirá completa com todos os serviços necessários.

🗄️ 5. Configuração do Banco de Dados

5.1. Opções de Hospedagem

Opção A: PostgreSQL Local

  • Instale PostgreSQL diretamente na VM
  • Use a connection string: postgresql://usuario:senha@localhost:5432/nome_db

Opção B: Azure Database for PostgreSQL

  • Solicite ao time a DATABASE_URL do banco hospedado na Azure
  • Exemplo: postgresql://usuario@servidor:senha@servidor.postgres.database.azure.com:5432/ctg_db
⚠️ Atenção: Se o banco estiver na Azure, é necessário garantir que a VM tenha acesso à porta 5432 do servidor PostgreSQL na Azure.

5.2. Testar Conexão com o Banco

# Testar conexão local
psql -U ctg_user -d ctg_engenharia

# Testar conexão remota (Azure)
psql "postgresql://usuario@servidor:senha@servidor.postgres.database.azure.com:5432/ctg_db"

🌐 7. Portas e Rede

As seguintes portas devem estar abertas na VM:

Porta Serviço Protocolo Uso
8080 Aplicação Web TCP Acesso à interface web
5432 PostgreSQL TCP Conexão com banco de dados

7.1. Configurar Firewall (Linux)

# Usando UFW (Ubuntu/Debian)
sudo ufw allow 8080/tcp
sudo ufw allow 5432/tcp
sudo ufw reload

# Verificar status
sudo ufw status
# Usando firewalld (CentOS/RHEL)
sudo firewall-cmd --permanent --add-port=8080/tcp
sudo firewall-cmd --permanent --add-port=5432/tcp
sudo firewall-cmd --reload

# Verificar status
sudo firewall-cmd --list-all
ℹ️ Nota: Se estiver usando Azure VM, configure também o Network Security Group (NSG) para permitir tráfego nessas portas.

🚀 8. Execução da Aplicação

8.1. Iniciar Backend

cd backend

# Modo produção
npm start

# Modo desenvolvimento (com hot-reload)
npm run dev

O backend iniciará em http://localhost:3001

8.2. Iniciar Frontend

cd frontend

# Modo desenvolvimento
npm run dev

# Build para produção e preview
npm run build
npm run preview

O frontend ficará disponível em http://localhost:5173 (dev) ou http://localhost:4173 (preview)

8.3. Acesso Final

✅ Aplicação disponível em:
  • Frontend: http://SEU_IP:8080
  • Backend API: http://SEU_IP:3001/api
  • Login Admin: Use as credenciais definidas no .env

🔍 9. Troubleshooting

Problema Causa Provável Solução
Erro de conexão com banco DATABASE_URL incorreta Verificar credenciais e conectividade
Porta já em uso Outro processo usando a porta lsof -i :3001 e matar o processo
Erro de CORS Origem não permitida Configurar CORS no backend
JWT inválido JWT_SECRET alterado Manter o mesmo secret ou relogin
Dependências não instaladas npm install não rodado Executar em backend e frontend

tail -f backend/logs/app.log

# Verificar portas em uso
netstat -tulpn | grep LISTEN

# Testar API
